# Roadmap

## Now

Current priorities:

- crawler stability
- analytics enhancement
- report quality
- documentation clarity

These tasks should improve the current deterministic workflow without changing the graph shape.

## Next

Near-term follow-up work after the current stage stabilizes:

- provider extensibility
- stronger contracts and schemas
- testing improvements

This phase should strengthen implementation quality without introducing orchestration complexity.

## Later

Deferred future work:

- LLM integration
- multi-LLM debate
- Feishu human-in-the-loop
- API orchestration
- task orchestration
- resumable execution

These items should only start after the core workflow, analytics, and documentation are stable enough to support expansion.
